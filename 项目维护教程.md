# OpenClaw 教程站维护教程

这份文档说明以后如何新增文章、如何放演示案例、文件夹名字能不能改，以及如何重新部署到 GitHub / Cloudflare Pages。

## 一、项目里文章放在哪里

所有正式教程文章都放在：

```text
source/_posts/OpenClaw教程/
```

当前推荐结构是“每篇文章一个目录”：

```text
source/_posts/OpenClaw教程/文章目录名/
├── index.md
├── 封面图.png
├── 01_xxx.png
└── 02_xxx.png
```

文章正文写在 `index.md` 里，图片直接放在同一个文章目录或子目录里。

示例：

```text
source/_posts/OpenClaw教程/OpenClaw网页设计open-design-artifacts/
├── index.md
└── 封面图.png
```

正文中引用图片：

```markdown
![步骤截图](./01_xxx.png)
```

封面图写法：

```yaml
cover: ./封面图.png
```

如果图片在子目录里，例如：

```text
assets/01-install-skill.png
```

正文可以写：

```markdown
![](assets/01-install-skill.png)
```

## 二、每次新增文章怎么做

推荐用脚本创建模板：

```bash
pnpm run new:tutorial -- <文章目录名> <分类名> <固定链接> <文章标题>
```

示例：

```bash
pnpm run new:tutorial -- OpenClaw新教程 网页设计 openclaw-new-tutorial OpenClaw 新教程标题
```

它会创建：

```text
source/_posts/OpenClaw教程/OpenClaw新教程/index.md
```

然后你只需要：

1. 编辑 `index.md`。
2. 把封面图放进同目录，命名为 `封面图.png`。
3. 把正文截图放进同目录或 `assets/` 子目录。
4. 检查 front matter。

文章 front matter 示例：

```yaml
---
title: OpenClaw 网页设计教程：用 open-design-artifacts 生成网站首页
date: 2026-05-10 22:30:00
updated: 2026-05-10 22:30:00
categories: 网页设计
tags:
  - OpenClaw
  - 网页设计
  - open-design-artifacts
abbrlink: openclaw-web-design-artifacts
cover: ./封面图.png
---
```

## 三、文件夹名字可以改吗

可以改，但要分清“源文件夹名”和“文章网址”。

文章源文件夹：

```text
source/_posts/OpenClaw教程/OpenClaw网页设计open-design-artifacts/
```

这个文件夹名字主要方便你管理文章，可以改。

真正决定文章访问地址的是 `index.md` 里的：

```yaml
abbrlink: openclaw-web-design-artifacts
```

因为 `_config.yml` 里配置了：

```yaml
permalink: posts/:abbrlink.html
```

所以文章最终网址是：

```text
/posts/openclaw-web-design-artifacts.html
```

结论：

- 只改文章文件夹名，不改 `abbrlink`：文章网址不变。
- 改 `abbrlink`：文章网址会变，旧链接会失效。
- 不要手动修改 `public/posts/`，那里是构建生成结果。

## 四、案例演示网站放在哪里

演示网站统一放在：

```text
source/demos/
```

每个案例一个目录：

```text
source/demos/sofa-brand/index.html
source/demos/openclaw-skill-market/index.html
source/demos/lumiere-investor-deck/index.html
```

访问地址分别是：

```text
/demos/sofa-brand/
/demos/openclaw-skill-market/
/demos/lumiere-investor-deck/
```

这些文件不会套博客主题，因为 `_config.yml` 里配置了：

```yaml
skip_render:
  - demos/**
```

这样 Hexo 会把演示站原样复制到 `public/demos/`。

## 五、案例文件夹名字可以改吗

可以改。

例如把：

```text
source/demos/sofa-brand/
```

改成：

```text
source/demos/lumiere-sofa/
```

访问地址也会变成：

```text
/demos/lumiere-sofa/
```

改名后必须同步更新文章里的链接，例如：

```html
<a href="/demos/lumiere-sofa/" target="_blank" rel="noopener">
```

如果不更新文章链接，用户会点到 404。

## 六、怎么把案例链接加到文章顶部

在文章 `index.md` 开头位置加入：

```html
## 案例预览

<div class="demo-link-grid">
  <a href="/demos/sofa-brand/" target="_blank" rel="noopener">
    <span>案例 1</span>
    <strong>沙发品牌设计网站</strong>
    <em>高级家居品牌首页</em>
  </a>
  <a href="/demos/openclaw-skill-market/" target="_blank" rel="noopener">
    <span>案例 2</span>
    <strong>OpenClaw 技能市场官网</strong>
    <em>暗色科技感产品发布页</em>
  </a>
</div>
```

样式已经写在：

```text
source/css/site-custom.css
```

## 七、怎么本地预览

固定使用 4000 端口：

```bash
pnpm exec hexo server -p 4000
```

访问：

```text
http://localhost:4000/
```

如果需要后台运行，可以用：

```bash
screen -dmS ovov-site-4000 bash -lc 'cd /Users/renyang/Desktop/ovov-site && pnpm exec hexo server -p 4000'
```

停止后台服务：

```bash
screen -S ovov-site-4000 -X quit
```

注意：改 `index.md` 通常会热更新；改 `_config.yml`、`scripts/`、主题配置时，需要重启 4000 服务。

## 八、怎么构建检查

每次发到线上前，先跑：

```bash
pnpm run clean && pnpm run build
```

构建后检查：

```text
public/posts/
public/demos/
```

重点确认：

- 新文章是否生成到 `public/posts/<abbrlink>.html`
- 文章图片是否生成到 `public/posts/<abbrlink>/`
- 演示案例是否生成到 `public/demos/<案例目录>/index.html`
- 不应该出现 `public/posts/0.html`

## 九、怎么重新部署

当前项目连接了 GitHub：

```text
https://github.com/yangbenyang/ovov-site.git
```

Cloudflare Pages 会从 GitHub 仓库重新部署。

完整流程：

```bash
pnpm run clean && pnpm run build
git status --short
git add -A
git commit -m "更新教程内容"
git push origin main
```

推送成功后，Cloudflare Pages 通常会自动触发部署。

如果 Cloudflare 没自动部署：

1. 打开 Cloudflare Dashboard。
2. 进入 Workers 和 Pages。
3. 找到 `ovov-site`。
4. 进入部署记录。
5. 点击 Retry deployment 或重新部署最新提交。

## 十、常见问题

### 1. 文章图片本地看不到

先检查写法：

```markdown
![](./图片名.png)
![](assets/图片名.png)
```

再检查文章是否有：

```yaml
abbrlink: xxx
```

最后重新构建：

```bash
pnpm run clean && pnpm run build
```

### 2. 演示页被套了博客主题

检查 `_config.yml` 是否有：

```yaml
skip_render:
  - demos/**
```

演示页必须放在：

```text
source/demos/
```

不要放到 `source/_posts/` 下面。

### 3. 改了案例目录名后链接打不开

同步修改文章里的 `/demos/xxx/` 链接。

### 4. 不想改文章网址

不要改 `abbrlink`。

文件夹名可以改，`abbrlink` 保持不变即可。
